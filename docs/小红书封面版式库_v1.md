# 小红书封面版式库 v1（18式）

> 用途：供"封面工厂"调用。技术路线：背景层（图像模型生成/产品实拍/纯色渐变）+ 文字层（HTML模板排版）→ Playwright/Chromium 渲染出图。AI直接生成中文文字图不可用，所有文字一律走HTML层。
> 工作流：质检环按选题类型从本库选 2 个版式 → 各渲染 1 张 → 视觉模型按文末《封面质检checklist》评分 → 高分者交付，低分打回换版式重渲。

---

## 全局规范（所有版式共用）

- **画布**：1242 × 1660 px（3:4，小红书主流竖版）。备选 1242 × 1242（1:1，多图笔记内页）。
- **安全区**：四边各留 8% 内边距；右下角 12% 区域避开关键信息（信息流角标遮挡位）。
- **主标题**：≤9个字为佳，硬上限12字；字号 ≥ 画布高度的 1/12（约 140px），行数≤2。
- **信息层级**：主标题 : 副标题 : 标签字 ≈ 3 : 1.5 : 1，层级数≤3，任何封面元素总数≤6。
- **字体**（免费可商用，服务端渲染必须自带字体文件）：思源黑体 Heavy/Bold（主力）、阿里巴巴普惠体（正文感）、站酷快乐体（活泼场景）、霞鹜文楷（手写感）。禁止使用未授权商用字体（微软雅黑、苹方均不可商用分发）。
- **对比度**：主标题与底色对比度 ≥ 4.5:1；浅色底配深字+可选描边，图片底必须加蒙层（黑/白 30~50% 透明度）再压字。
- **三米法则**：缩略到 25% 尺寸（模拟信息流）主标题仍可读，质检环用 310px 宽缩略图复检。

## 第一组 · 大字报系（干货/观点类首选，制作成本最低）

**V01｜纯色大字报**
- 结构：纯色或双色渐变底；主标题居中偏上（垂直40%位）；副标题其下；底部小字标签条。
- 适用：观点输出、清单干货、教程总纲
- 文字层：主标题≤9字可换行强调色高亮2~3个关键字；副标题≤16字。
- HTML要点：背景 `linear-gradient(160deg, 色A, 色B)`；关键字用 `<span>` 换色或加底色块；高亮色与底色互补。
- 质检点：高亮关键字是否恰好是"钩子词"；纯色底是否单调（可加 5% 噪点纹理或几何装饰块）。

**V02｜人像底大字报**
- 结构：人物/场景图铺满；底部40%叠深色渐变蒙层；主标题压在蒙层上；左上角小logo/栏目名。
- 适用：IP人设号、职场、情感观点
- HTML要点：`background-image` + `linear-gradient(transparent 45%, rgba(0,0,0,.65))` 双层背景；文字 `text-shadow` 兜底。
- 质检点：人脸不被文字遮挡；蒙层后文字对比度达标。

**V03｜数字爆炸式**
- 结构：超大数字为视觉主体（占画面高度40%+），数字旁缀小单位字；主标题环绕数字上/下方。
- 适用：价格锚点（"9.9元"）、时间承诺（"3天"）、数量盘点（"17个"）
- HTML要点：数字字号 600px+，用 Bold 数字字体；单位字号为数字的 1/4；数字可加描边或投影制造体积感。
- 质检点：数字是否是该选题真正的钩子（不是就换版式）。

**V04｜手写便签式**
- 结构：米色/牛皮纸纹理底；中央白色"便签卡"带轻投影；便签内手写体标题+2~3行要点；可加胶带贴角装饰。
- 适用：经验贴、自留笔记感干货、学习类
- HTML要点：便签 `box-shadow: 0 8px 24px rgba(0,0,0,.12)`；旋转 `-1.5deg` 增加手帐感；字体用霞鹜文楷。
- 质检点：手写体在25%缩略下可读性（不行则主标题换黑体、仅装饰字用手写体）。

## 第二组 · 对比系（测评/改造类首选，点击率天花板）

**V05｜上下对比**
- 结构：上下分屏各50%；中缝VS胶囊条或闪电分割线；上屏"Before/雷"标签，下屏"After/真香"标签；主标题压中缝。
- 适用：前后改造、产品对比、错误vs正确示范
- HTML要点：两屏各自独立 `background-image`；中缝标题加白色描边+投影保证跨两屏可读。
- 质检点：两屏明暗/色调反差是否足够（反差即点击率）。

**V06｜左右对比**
- 结构：左右分屏；顶部通栏主标题；两屏底部各一行结论小字（如"99元 vs 999元"）。
- 适用：平替对比、双品PK、二选一裁决
- 质检点：左右主体物大小、角度是否一致（不一致显得不公平，评论区会骂）。

**V07｜红黑警告式**
- 结构：高对比配色（黑底+警示黄/红）；顶部⚠️图标或"避坑"印章；主标题大字；下方2~3条雷点预览（划线删除样式）。
- 适用：避坑、维权、防骗类
- HTML要点：印章用CSS圆框+旋转-12°+粗边；雷点条用 `text-decoration: line-through` 或前缀❌。
- 质检点：不触发平台"引人不适"审核（血腥/恐吓视觉禁用，警示靠配色不靠惊悚图）。

## 第三组 · 教程清单系（搜索/收藏向首选）

**V08｜封面即目录**
- 结构：顶部30%主标题区；下方70%列出3~5条要点（带序号①②③，每条≤12字），底部"完整版见正文"小字。
- 适用：长干货、合集、攻略
- HTML要点：要点行用 flex 列表，序号用强调色圆形角标；行间距 = 字号 × 1.8。
- 质检点：要点是否"半露"（全说完没人点进来，要点写结论的前半句）。

**V09｜步骤四宫格**
- 结构：2×2网格，每格一张步骤缩略图+左上角步骤序号角标；网格上方通栏主标题。
- 适用：实操教程（化妆/做饭/软件操作）
- HTML要点：grid 布局 gap 12px；序号角标用强调色方块，格内图片 `object-fit: cover`。
- 质检点：四格图视觉风格统一（色温一致），步骤逻辑从左上到右下。

**V10｜界面演示式**
- 结构：手机/电脑界面截图为主体（占60%），关键位置红框+箭头标注；顶部主标题；截图下一行结果小字。
- 适用：软件教程、薅羊毛操作、后台数据展示
- HTML要点：截图加圆角+描边模拟设备感；标注框 `border: 4px solid red` + 手绘风箭头SVG。
- 质检点：截图内敏感信息打码（昵称/金额/手机号），红框是否框在真正的关键位。

**V11｜表格对比式**
- 结构：主标题在顶部；中部一张3~5行的简化对比表（列：维度/A/B），胜出项打✓高亮。
- 适用：多品横评、参数对比、选购决策
- HTML要点：表格行高≥90px，表头用强调色底；✓用大号绿色，✗用灰色（别用红色，红色留给价格）。
- 质检点：表格≤5行（多了缩略图看不清，完整表放正文）。

## 第四组 · 种草展示系（电商/好物类首选）

**V12｜产品特写+卖点飘带**
- 结构：产品大图居中偏右；左侧纵向2~3条卖点短语（每条带icon）；顶部主标题；右下价格角标。
- 适用：单品种草
- HTML要点：卖点条用半透明白底胶囊；价格角标用爆炸贴样式（CSS clip-path 星形）。
- 质检点：产品图抠图边缘干净（建议背景层直接用产品白底图+渐变底色合成）。

**V13｜多品陈列+价签**
- 结构：3~6个产品平铺陈列（网格或错落），每个旁缀小价签；顶部主标题（常配公式10"X元搞定Y"）。
- 适用：合集种草、好物清单、店铺挖宝
- 质检点：产品数量与标题数字一致（标题说8个图里只有5个=差评）。

**V14｜九宫格氛围拼图**
- 结构：3×3图片拼贴铺满；中央格替换为色块+主标题。
- 适用：穿搭合集、家居灵感、旅行相册类
- HTML要点：grid 无gap或2px细缝；中央格底色取九图主色调的对比色。
- 质检点：九图色调统一（提前用视觉模型按色温聚类筛图——这就是"图片筛选"能力的落点之一）。

**V15｜截图证据流**
- 结构：聊天记录/订单/收益截图为主体，关键句红线划出；顶部主标题；截图轻微旋转叠放2~3层制造"一堆证据"感。
- 适用：副业晒单、避坑实录、服务见证
- 质检点：截图真实性存档（合规底线：员工只排版用户提供的真实截图，绝不生成伪造聊天记录/收益图——此为硬禁令，写入员工工作纪律）。

## 第五组 · 风格系（人设/品牌向）

**V16｜杂志封面式**
- 结构：人物半身图为主体；顶部大号刊头字（栏目名/IP名）；左右两侧散布3~4条小标题（杂志coverline样式）；底部期号小字。
- 适用：IP人设号、访谈、周更栏目
- HTML要点：刊头字号介于主副标题之间但字重最大；coverline 错落对齐左右边缘。
- 质检点：小标题≤4条，多了变垃圾杂志。

**V17｜问答悬念式**
- 结构：大面积留白或虚化底；中央一个巨大"？"或问题主标题；底部一行"答案在第3张"引导。
- 适用：互动钩内容、冷知识、争议话题
- 质检点：问题必须是目标人群真实想知道的（质检环用"会有人搜这个问题吗"判定）。

**V18｜前情提要剧透式**
- 结构：结果画面铺底（如退货单/成品图）；主标题用结局句（配合标题公式05"用了三个月，我把X退了"）；左上角"完整经过👉"小贴纸。
- 适用：测评反转、故事型笔记
- 质检点：封面结局与正文结论一致（标题党但不诈骗，骗点击的代价是掉粉+限流）。

## 封面质检 checklist（视觉模型逐项判定，输出 通过/打回+原因）

1. 25%缩略图下主标题可读？
2. 主标题≤12字、≤2行、元素总数≤6？
3. 文字与底图对比度达标（含蒙层后）？
4. 安全区无关键信息越界、右下角标位干净？
5. 版式与选题类型匹配（对照本库"适用"字段）？
6. 图片层质量：无AI畸变手指/乱码文字、人脸完整、产品主体清晰？
7. 合规：无违禁词上图、无伪造证据图、截图已打码？
8. 风格连续性：与该账号最近9篇封面同一视觉体系（主色、字体、版式族——账号主页九宫格整体观感也是转化项）？
